# 마이링크 (MyLink) - 사용자 시나리오 (User Scenario) v1.1

본 문서는 `PRD v1.5`를 바탕으로 작성되었으며, 사용자의 구체적인 행동 흐름을 "사용자는 __ 하기 위해 __ 한다" 형식으로 정의합니다.

---

## 1. 방문자 (Visitor) : 마이링크 접속 및 이용 과정

### 1-1. 접속 및 프로필 확인
- 방문자는 **사용자의 여러 외부 채널을 확인하기 위해** 소셜 미디어 등에 공유된 URL(`도메인/닉네임`)을 **클릭하여 접속한다.**
- 방문자는 **이 페이지의 주인이 누구인지 식별하기 위해** 페이지 상단의 `Username`(실명), `displayName`(닉네임), 그리고 **소개글을 읽는다.**

### 1-2. 링크 탐색 및 이동
- 방문자는 **자신이 원하는 목적지(유튜브, 쇼핑몰 등)로 이동하기 위해** 나열된 링크 블록들 중 하나의 버튼을 **클릭한다.**
- 방문자는 **창작자의 다른 링크도 추가로 둘러보기 위해** 새 창으로 열린 외부 목적지 창을 닫고 본래의 마이링크 랜딩 페이지로 **다시 돌아온다.**

---

## 2. 소유자 (Owner) : 위지윅(WYSIWYG) 기반 관리 과정

> **💡 핵심 UX (위지윅 뷰)**: 소유자가 로그인하면, 별도의 복잡한 '관리자 설정 페이지' 없이 방문자가 보는 화면과 **동일한 형태의 화면**이 노출됩니다. 소유자는 화면 위에서 대상을 직접 클릭해 고치게 됩니다.

### 2-1. 관리 시작 및 빈 상태 (Empty State) 처리
- 소유자는 **자신의 링크 정보를 최신화하기 위해** 메인 화면에서 구글 소셜 로그인 버튼을 **클릭하여 접속한다.**
- (가입 직후) 소유자는 **처음 화면에서 무엇을 해야 할지 가이드를 받기 위해** "첫 번째 링크를 추가해보세요!" 라고 적힌 **초기 안내 문구(Empty State)를 확인한다.**

### 2-2. 링크 추가 과정 (최상단 노출)
- 소유자는 **새로운 방문 목적지 URL을 추가하기 위해** 화면 상단에 있는 **'[링크 추가]' 버튼을 클릭한다.**
- 소유자는 **가장 최근에 추가한 내용을 즉시 세팅하기 위해** 새롭게 생성된 '제목 없음, URL 없음' 상태의 빈 블록이 링크 리스트의 **맨 위(최상단)에 나타나는 것을 확인한다.**

### 2-3. 인라인 편집 및 URL 엣지 케이스 처리
- 소유자는 **프로필이나 링크 내용을 변경하기 위해** 수정 화면으로 이동하지 않고, 현재 보이는 텍스트나 URL 영역을 마우스로 **직접 클릭한다.**
- 소유자는 **내용을 최신화하기 위해** 클릭된 영역 내의 기존 텍스트를 지우고 새로운 내용을 **타이핑한다.**
- 소유자는 **수정한 내용을 가장 빠르게 저장하기 위해** 텍스트를 다 친 후 키보드의 **`Enter` 키를 눌러 수정을 완료(DB 저장)한다.**
- 소유자는 **일일이 프로토콜을 입력하는 수고를 덜고 에러를 방지하기 위해**, 목적지 URL 입력 창에 `google.com`만 치고 Enter를 누르더라도 시스템이 자동으로 앞에 `https://`를 **붙여서 저장해주는 것을 경험한다.**

### 2-4. 링크 삭제 과정 및 안전 장치
- 소유자는 **더 이상 유효하지 않은 특정 링크를 없애기 위해** 해당 링크 우측에 위치한 '휴지통' 아이콘을 **클릭한다.**
- 소유자는 **단순 클릭 실수로 멀쩡한 데이터가 날아가는 것을 막기 위해** 화면에 나타난 "정말 삭제하시겠습니까?" 경고 팝업 창에서 **'확인' 버튼을 누른다.**
