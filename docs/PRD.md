# 마이링크 (MyLink) - PRD (제품 기능 정의서) 초안 v1.5

## 1. 프로젝트 개요 (Overview)
- **프로젝트 명**: 마이링크 (MyLink)
- **대상 사용자**: 
  - 여러 채널(SNS, 포트폴리오 등)을 운영하여 본인만의 통합 링크 페이지가 필요한 개인, 소상공인, 크리에이터
- **목적**: 
  - 복잡한 절차 없이 아주 가볍고 직관적으로 자신만의 '멀티 링크 랜딩 페이지' 구축
  - 사용자의 닉네임을 기반으로 하는 고유한 URL(`도메인/[닉네임]`)로 쉽게 링크 공유
  - 복잡한 기능(과금, 테마, 통계 등)을 배제한 본질적인 '정보 연결'에 집중

---

## 2. 필수 기능 목록 (Core MVP Features)

본 프로젝트는 추후 고도화 내용을 배제하고, 당장 완성해야할 가장 직관적이고 필수적인 기능(MVP)에 집중합니다.

1. **단일 로그인 (Google 전용)**: 회원 가입 창이나 정보 입력 등 복잡한 절차 없이 구글 소셜 로그인만 제공. `displayName` 초기값 자동 생성.
2. **프로필 정보 관리 (인라인 편집)**: 메인 상단에 표시되는 **`Username`**(실제 이름), **`displayName`**(닉네임 등 표시 이름), 그리고 **소개글** 편집 기능. (프로필 사진 없음)
3. **직관적인 링크 관리 (인라인 편집)**: 무제한 링크 추가, 수정, 삭제. (링크 숨김 On/Off 기능 제외)
4. **닉네임 기반 URL 라우팅**: 핸들 대신 사용자의 닉네임(`displayName`) 텍스트가 개인 접속 URL 주소가 됩니다.

---

## 3. 각 기능의 상세 구조 (Detailed Specifications)

### 1. 구글 소셜 로그인 전용 (Google OAuth) 및 초기화
- 이메일 인증이나 비밀번호 찾기 모듈을 생략하고 **Google OAuth 소셜 로그인** 단일 체제를 적용합니다.
- 계정 최초 생성(초기 로그인) 시, 사용자의 구글 지메일 주소 앞부분(예: `abc@gmail.com`의 경우 `abc`)을 파싱하여 **`displayName`의 초기값으로 자동 할당**합니다.

### 2. 프로필 UI 및 텍스트 인라인 편집
- 페이지 상단부에는 복잡한 이미지 영역 없이 사용자의 **`Username`**(실명), **`displayName`**(닉네임)과 **소개 텍스트**가 간결하게 표시됩니다.
- 본인 페이지에 접속한 계정 주인은 표시되는 세 가지 텍스트를 마우스로 직접 클릭하여 바로 그 자리에서 수정할 수 있는 **인라인 편집(Inline Editing)**이 적용됩니다. 

### 3. 무제한 링크 생성 및 관소화된 관리 방식
- 지정된 목적지(URL) 주소와 버튼 위 텍스트를 제한 없이 계속해서 추가할 수 있습니다.
- 등록된 링크를 수정할 때에도 별도의 수정 팝업 창을 띄우지 않고, 화면상의 글꼴을 직접 눌러서 내용과 URL을 고치는 **인라인 편집**을 적용합니다.
- 복잡성을 줄이기 위해 토글 스위치(On/Off 상태 전환) 기능을 과감히 제외하고 '추가', '수정', '영구 삭제'만을 다룹니다.

### 4. 고유 URL 발급 및 데이터베이스 모델링
- 별다른 기호나 영어 강제 조합 없이, 사용자가 자유롭게 구성한 닉네임(`displayName`) 정보가 곧 방문자용 접속 URL 주소가 됩니다. (접속 접속 예: `mylink.com/홍길동`)
- **DB 아키텍처**: 데이터베이스는 계정 메인 Document 하위에 `Username`, `displayName`, `소개글`을 보관하며, 다수의 링크 정보 객체들을 저장하기 위한 **서브 컬렉션(Sub-collection)** 구조를 채택하여 NoSQL 구조에 맞게 모델링합니다.
